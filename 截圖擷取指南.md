擷取 5 張 1242x2688 解析度的截圖,提升 PWABuilder 分數從 16/44 到 30-34/44,達到上架標準。

## 步驟 1: 啟動開發伺服器

開啟終端機並執行:

```bash
npm run dev
```

等待伺服器啟動,通常會顯示類似:
```
VITE v5.x.x  ready in xxx ms

➜  Local:   http://localhost:5173/
➜  Network: use --host to expose
```

保持此終端機運行,**不要關閉**。

## 步驟 2: 開啟 Chrome DevTools 裝置模擬

1. **開啟 Chrome 瀏覽器**,訪問 `http://localhost:5173/`

2. **開啟 DevTools**:
   - 按 `F12` 鍵,或
   - 右鍵點擊頁面 → "檢查"

3. **切換到裝置模擬模式**:
   - 按 `Ctrl + Shift + M` (Windows/Linux)
   - 或點擊 DevTools 左上角的 📱 手機圖標

4. **選擇裝置** (重要!):

   **方法 1: 直接選擇 iPhone 14 Pro Max (最簡單)**
   - 在頂部裝置下拉選單中,選擇 **"iPhone 14 Pro Max"**
   - Chrome 會自動設定正確的尺寸和 DPR
   - 您應該會看到尺寸顯示為 430 x 932

   **方法 2: 手動設定 Responsive (如果找不到 iPhone 14 Pro Max)**
   - 選擇 **"Responsive"**
   - 在頂部工具列,您會看到寬度和高度的輸入框
   - 設定為: **430** x **932**
   - 在尺寸旁邊,點擊 **"⋮" (三個點)** 或 **齒輪圖標**
   - 找到 **"Add device pixel ratio"** 或直接在下拉選單中選擇縮放比例
   - 如果看到百分比 (100%, 75%, 50%),改選 **"100%"** 或 **"Fit to window"**

5. **如果找不到 DPR 設定,請使用以下替代方案**:

   **替代方案: 使用瀏覽器縮放功能**
   - 選擇 "Responsive" 模式
   - 設定尺寸為 **1242** x **2688** (直接設定最終尺寸)
   - 這樣截圖就會直接是 1242x2688,不需要後續調整

   **或者: 直接選擇預設的高解析度裝置**
   - 在裝置列表中選擇: **"iPhone 12 Pro"**, **"iPhone 13 Pro"**, 或 **"Pixel 5"**
   - 這些裝置的 DPR 都是 2-3,截圖解析度會較高

## 步驟 3: 擷取 5 張關鍵截圖

您需要擷取以下 5 個頁面的截圖:

### 截圖 1: 上傳作業頁面 ✅

**擷取方式**:

4. 輸入 "screenshot"

6. 截圖會自動下載到您的下載資料夾

**檔案命名**: 將下載的檔案重新命名為 `screenshot-1-upload.png`

- 使用圖片編輯軟體模擬相機介面 (不推薦)

**擷取方式**: 同上

**檔案命名**: `screenshot-2-camera.png`

## 步驟 5: 調整截圖尺寸為 1242x2688

如果您的截圖不是 1242x2688,需要調整尺寸。

### 方法 1: 使用 sharp-cli (推薦)

開啟終端機,進入截圖所在資料夾 (通常是 `C:\Users\您的使用者名稱\Downloads`):

```bash
cd C:\Users\GPPS\Downloads
```

調整每張截圖:

```bash
npx sharp-cli resize 1242 2688 -i screenshot-1-upload.png -o screenshot-1-upload-resized.png --fit cover
npx sharp-cli resize 1242 2688 -i screenshot-2-camera.png -o screenshot-2-camera-resized.png --fit cover
npx sharp-cli resize 1242 2688 -i screenshot-3-grading.png -o screenshot-3-grading-resized.png --fit cover
npx sharp-cli resize 1242 2688 -i screenshot-4-result.png -o screenshot-4-result-resized.png --fit cover
npx sharp-cli resize 1242 2688 -i screenshot-5-gradebook.png -o screenshot-5-gradebook-resized.png --fit cover
```

完成後,刪除原始檔案,將 `-resized.png` 檔案重新命名回原檔名。

### 方法 2: 使用線上工具

如果不想用指令,可以使用:
- https://www.iloveimg.com/resize-image
- 上傳截圖
- 選擇「自訂尺寸」
- 輸入: 寬度 1242, 高度 2688
- 選擇「不保持比例」或「裁切以填滿」
- 下載

## 常見問題

### Q1: 我沒有登入帳號,無法看到批改清單和成績頁面怎麼辦?

**解決方案**:
1. 在本地環境註冊一個測試帳號
2. 建立 1-2 個測試作業
3. 完成 1 個作業的批改流程
4. 這樣就能截到有內容的頁面

**替代方案**: 如果無法建立測試資料,可以擷取:
- 空白狀態頁面 (顯示「尚無作業」的畫面)
- 其他公開頁面 (例如首頁、功能介紹頁面)

### Q2: Chrome DevTools 的 "Capture full size screenshot" 找不到?

**原因**: 您可能沒有切換到裝置模擬模式。

**解決方案**:
1. 確保按了 `Ctrl + Shift + M` 或點擊了 📱 圖標
2. 頂部應該會出現裝置選單 (Responsive / iPhone / iPad 等)
3. 再次按 `Ctrl + Shift + P` 應該就能看到 "Capture full size screenshot"

### Q3: 截圖解析度太低 (只有 430x932) 怎麼辦?

**最簡單的解決方案**: 直接將 Responsive 模式的尺寸設定為 **1242 x 2688**

**步驟**:
1. 在 DevTools 裝置模擬模式中,選擇 "Responsive"
2. 在頂部的寬度/高度輸入框中,輸入:
   - 寬度: **1242**
   - 高度: **2688**
3. 按 Enter 確認
4. 使用 `Ctrl + Shift + P` → "Capture full size screenshot"
5. 截圖會直接是 1242x2688,完全不需要調整!

**這是最可靠的方法**,因為它不依賴 DPR 設定。

### Q4: 我可以使用 Windows 內建的截圖工具 (Win + Shift + S) 嗎?

**不推薦**,因為:
- 可能無法擷取完整頁面 (只能截可見區域)
- 解析度可能不足

**建議**: 使用 Chrome DevTools 的 "Capture full size screenshot" 功能,它會:
- 自動擷取整個頁面 (包含捲動區域)
- 保留正確的解析度
- 符合裝置模擬設定
