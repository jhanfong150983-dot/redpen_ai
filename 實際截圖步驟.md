# RedPen AI 截圖實際步驟

## 重要發現
您的應用程式需要**先登入**才能訪問大部分功能頁面。因此我們有兩個方案:

---

## 方案 A: 使用登入後的頁面 (推薦,展示完整功能)

### 步驟 1: 準備測試帳號

1. 啟動開發伺服器:
```bash
npm run dev
```

2. 訪問 `http://localhost:5173/`

3. **登入或註冊**一個測試帳號

4. 登入後,您會看到主頁,有以下功能區塊:
   - 班級管理
   - 作業管理
   - 作業匯入
   - AI 批改
   - 訂正管理
   - 成績管理

---

### 步驟 2: 設定 Chrome DevTools

1. 按 `F12` 開啟 DevTools

2. 按 `Ctrl + Shift + M` 切換到裝置模擬模式

3. 在頂部選擇 **"Responsive"**

4. **重要!** 直接設定尺寸為:
   - 寬度: **1242**
   - 高度: **2688**

這樣截圖就會直接是 1242x2688,不需要調整!

---

### 步驟 3: 擷取可用的頁面截圖

#### 截圖 1: 首頁/登入後主頁 ✅ (最重要!)

**訪問**: `http://localhost:5173/` (登入後)

**擷取方式**:
1. 按 `Ctrl + Shift + P`
2. 輸入 "screenshot"
3. 選擇 **"Capture full size screenshot"**

**檔案命名**: `screenshot-1-home.png`

**為什麼重要**: 這是使用者第一眼看到的畫面,展示所有核心功能!

---

#### 截圖 2: 班級管理 ✅

**訪問**: 在首頁點擊「班級管理」按鈕

**擷取方式**: 同上

**檔案命名**: `screenshot-2-classroom.png`

---

#### 截圖 3: 作業管理 ✅

**訪問**: 在首頁點擊「作業管理」按鈕

**擷取方式**: 同上

**檔案命名**: `screenshot-3-assignment.png`

---

#### 截圖 4: 補充墨水頁面 ✅

**訪問**: 在首頁右上角點擊「補充墨水」按鈕

**擷取方式**: 同上

**檔案命名**: `screenshot-4-ink-topup.png`

**為什麼有用**: 展示付費機制,對應用商店很重要!

#### 截圖 5: AI 批改清單 (如果有資料) ⚠️

**如果沒有資料**: 會顯示「尚無作業」的空白頁面,仍可截圖

## 方案 B: 使用公開頁面 (如果沒有測試帳號)

如果您不想建立測試帳號,可以只截取未登入的頁面:

### 截圖 1: 首頁/Landing Page ✅

**訪問**: `http://localhost:5173/` (未登入狀態)

這應該是您的行銷首頁,展示 RedPen AI 的介紹影片和功能說明。

**檔案命名**: `screenshot-1-landing.png`

### 如果只有 1 張截圖怎麼辦?

**別擔心!** 即使只有 1-2 張截圖,我們仍可以:
1. 重複使用同一張截圖 (PWABuilder 允許)
2. 調整 vite.config.ts 的 screenshots 為至少 2 張
3. 配合其他優化項目,仍可將分數提升到 24-28 分

**替代方案**: 使用圖片編輯軟體,製作簡單的功能展示截圖:
- 截圖 1: 首頁
- 截圖 2: 使用相同首頁,但裁切不同區域
- 截圖 3-4: 使用 Figma/Canva 製作簡單的功能說明圖

1. **登入您的測試帳號** (如果有)

3. **調整尺寸** (如果需要):
```bash

npx sharp-cli resize 1242 2688 -i screenshot-1-home.png -o screenshot-1-home-resized.png --fit cover
```

